## 1. 这份文档是做什么的

本页用于告诉不同读者“应该先看哪份文档”，避免业务用户、管理员、交付人员和研发人员拿错文档。

原则：

1. 业务用户优先看“怎么做”
2. 管理员优先看“怎么管”
3. 交付、联调与运维优先看“接口、架构与部署”
4. 如遇真实无人机、机巢 / 机场、视频工程化需求，先确认当前仍属于后续规划阶段

## 4. 当前能力边界说明

截至 `phase-39`：

- 平台产品化交付链路已完成
- 文档已同步到当前控制台形态
- 真实无人机 / 机巢 / 视频工程化仍属于后续 `phase-40` 至 `phase-43` 规划

因此：

1. 面向客户时，可以按当前文档做培训和交付说明
2. 但不要把后续规划中的真实接入能力当成“已经落地”的现成能力

## 5. 文档使用建议

1. 业务培训时，避免直接发 API 附录给普通用户。
2. 管理员培训时，先讲主路径，再讲高级治理区。
3. 如果文档内容与实际页面不一致，以当前系统页面和 `docs/PROJECT_STATUS.md` 为准，再回头修正文档。
