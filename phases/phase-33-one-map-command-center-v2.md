# Phase 33 - 一张图主界面与指挥中心 V2

## 0. Basis
- Based on: `项目最终目标.md`
- Depends on: `phases/phase-32-role-workbench-productization.md`

## 1. Objective
将地图提升为系统主交互面，形成真正的“一张图掌握空中力量”的日常值守与指挥主界面。

## 2. Scope
- 地图主视图替代分散式模块入口
- 统一图层：任务、设备、空域、告警、事件、成果
- 指挥中心 V2：值守模式、领导模式、演示模式
- 时间轴、轨迹回放、图层切换、事件联动卡片
- 地图对象快捷操作：从对象直接进入处置或详情

## 3. Out of Scope
- 3D 专业 GIS 引擎升级
- 真正的多路视频调度编排平台化增强

## 4. Deliverables
- 地图主界面与指挥大屏 V2
- 图层与时间轴交互体系
- 地图对象快捷动作入口
- 指挥/演示双模式演示脚本

## 5. Acceptance
- 调度员可在地图页完成主要监控和关键跳转
- 领导视图可直接展示态势、任务、告警与统计
- 日常值守不依赖频繁切换多个页面

## 6. Product Delivery Standard
- 本阶段新增或改造的 UI，默认按“正式交付给非技术业务人员长期使用”的标准设计，不允许回退到工程调试台风格
- 本阶段必须继承并保持 Phase 32 已达成的产品化效果：中文化、对象选择驱动、统一壳层、统一组件、统一状态反馈、管理员能力下沉，这些基线不得因地图化重构而回退
- 地图对象交互必须优先提供业务语义明确的动作入口，而不是暴露底层字段、对象 ID 或接口参数
- 页面必须延续统一中文文案、统一视觉组件和统一状态反馈，不得形成独立样式孤岛
- 任何未达到标准的旧页面，如被本阶段触达，必须同步改造或明确下沉，不得以“后续再说”方式留在主路径
- 任何对既有主业务页或管理员页的改动，都必须达到“不低于 Phase 32 当前完成效果”的标准；如新增交互替代旧交互，最终呈现必须整体更优而不是局部更优

## 7. Exit Criteria
- `ruff`, `mypy`, `pytest`, `e2e` 全绿
- 本阶段涉及的地图与指挥页面全部达到正式交付标准
- 本阶段触达的既有页面保持或优于 Phase 32 的产品化效果，不得出现交互、文案、导航或视觉回退
- 指挥中心 V2 演示脚本可复跑
